# 需求分析 — SkillS Manager

## 1. 项目背景

AI 编程助手（Agent）的 Skill 生态系统日益丰富，用户可能同时使用多个 Agent（如 opencode、Cursor 等），各自安装了不同的 Skill。缺乏统一的视图来查看和管理这些 Skill 的清单、来源和用法。

## 2. 用户目标

- 在一处查看本机所有 Agent 已安装的 Skill
- 了解每个 Skill 的来源、说明、用法和命令
- 支持简体中文翻译，降低阅读门槛
- 能够手动刷新，获取最新安装的 Skill 列表

## 3. 功能需求

| ID | 需求 | 说明 |
|----|------|------|
| F1 | 扫描 Skill 清单 | 扫描本机多个 Agent 的 Skill 安装目录，发现所有已安装 Skill |
| F2 | 汇总详细信息 | 每条 Skill 展示：来源 Agent、名称、说明、用法、命令 |
| F3 | 简体中文翻译 | 英文说明/用法同步显示中文翻译 |
| F4 | 手动刷新 | 用户点击按钮即可重新扫描并更新列表 |
| F5 | Web 页面展示 | 本地 Web 服务，浏览器查看 |

## 4. 非功能需求

- 启动后自动扫描一次
- 页面加载响应 < 2s
- 仅读取本地文件系统，不修改任何内容
